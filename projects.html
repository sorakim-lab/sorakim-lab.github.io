projects / regulatory UX / human judgment in regulated environments

How systems shape human judgment.

I study what I call Regulatory UX — the interaction experience of workers inside compliance-constrained systems, where procedural documents, audit workflows, and investigation structures organize navigation, cognition, and decision-making. These six projects are the empirical ground for that argument.

Two research axes, one argument: How do GMP systems shape reasoning? (Investigation axis) — and how do they shape navigation? (Access axis.) Every project examines one angle of that constraint — and every design response holds the same condition: compliance-safe, external, non-invasive.

Research statement

I study how regulatory systems shape human judgment — through document design, workflow structure, and investigation scaffolding — in compliance-constrained GMP environments. Six projects examine this across two axes: how workers reason under investigation pressure, and how they navigate procedural document environments.

Design condition

No validated source files are modified. All interventions are compliance-safe and external — support layers only.

Regulatory UX concept

The umbrella framework connecting all six projects — currently being developed as a standalone concept paper.

axis 01 · reasoning & investigation

How GMP systems shape causal reasoning

These two projects examine how investigation workflows constrain reasoning — through closure pressure, premature convergence, and accountability structures that compress judgment before the causal picture is complete.

reasoning · 01

RCA Reasoning Scaffolder

N = 15 paper-track

A human-centered scaffold for root cause analysis, designed to keep causal hypotheses open and resist the structural pressure toward premature accountability convergence that GMP investigation workflows produce.

reasoning design causal mapping PAC counter-design streamlit python

The problem

  • Root cause analysis (RCA) narratives in pharmaceutical quality investigations converge on individual blame — not because evidence demands it, but because document structure and closure pressure do.
  • Investigators lack a layer for maintaining and comparing alternative causal hypotheses before committing to formal documentation.

Results

N=15. 80% of participants increased causal breadth. All 5 who initially attributed causes to individual error revised toward systemic factors. 15/15 reported no premature accountability convergence after using the scaffold.

reasoning · 02

Premature Causal Convergence in RCA

N = 5 paper-track

A pilot study showing that pharmaceutical QC practitioners universally begin out-of-specification (OOS) deviation analysis with a single cause — and that closure pressure in investigation workflows truncates reasoning before systemic factors can surface.

premature causal convergence closure pressure pilot study time-based observation

The problem

  • In pharmaceutical quality practice, investigators spend less than 10 minutes on initial causal analysis before documentation begins — not by choice, but because investigation timelines structurally demand early closure.
  • Single-cause convergence at the outset is universal across practitioners, regardless of what type of cause they identify.

Results

N=5. 5/5 began with a single cause. 5/5 revised and expanded their causal model within 10 minutes when given time — confirming that the problem is structural, not cognitive.

axis 02 · navigation & access

How GMP systems shape document navigation

These four projects examine how procedural document environments constrain access, navigation, and review — through unsearchable archives, fragmented cross-references, opaque review cycles, and invisible navigation friction.

navigation · 01

SOP Accessibility in GMP Environments

N = 12 paper-track

A human-centered document UX project improving retrievability across scanned standard operating procedure (SOP) archives in a pharmaceutical quality control lab — demonstrating that usability gains can emerge through external support layers without modifying validated source files.

document UX information retrieval streamlit python usability study

The problem

  • Scanned procedure documents (SOPs) were stored as unsearchable PDFs — QC analysts spent 6–7 minutes per document retrieval using manual scrolling through hundreds of files.
  • Access failure is a judgment failure: workers who cannot find the right procedure at the right moment cannot apply it correctly.

Results

N=12. Retrieval time reduced from ~6–7 min to ~1–2 min. 4.83/5 usability. 100% adoption intent. Deployed at sora-sop-demo.streamlit.app.

navigation · 02

Cross-Reference System for GMP Documentation

N = 17 paper-track

A metadata-driven cross-reference system improving document traceability and reducing cognitive load across deviation reports, corrective and preventive action (CAPA) records, change controls, and related pharmaceutical quality documentation.

traceability metadata mapping streamlit python network analysis

The problem

  • Validated quality records (deviation reports, CAPAs, change controls) could not be automatically interlinked — practitioners manually cross-referenced record IDs across fragmented systems.
  • Version inconsistency and lookup burden weakened both daily work and regulatory inspection readiness.

Results

N=17. Task completion time reduced 30–60%. Documents discovered per session increased 2–3×. Path completion rate up from 30–50% to 70–90%. 17/17 adoption intent.

navigation · 03

Cognitive Navigation Explorer

N = 5 prototype study

A behavioral tracking prototype that records how GMP practitioners navigate procedural documents — capturing loopbacks, cross-reference jumps, and hesitation patterns as structured data.

document navigation loopback tracking behavioral data streamlit python

The problem

  • Navigation friction in pharmaceutical procedural documentation is invisible — when practitioners loop back or hesitate between documents, that behavior leaves no trace in current quality systems.
  • Without behavioral data, it is impossible to distinguish interface usability problems from document structure problems.

Results

N=5. 5/5 exhibited loopback behavior. 5.0/5 usability across all dimensions. Less experienced practitioners identified standard operating procedure (SOP) structure — not the interface — as the source of navigation confusion. 100% adoption intent.

navigation · 04

Smart Feedback & Review Dashboard

N = 6 prototype study

A compliance-safe workflow observability prototype visualizing reviewer workload, feedback patterns, and latency bottlenecks in deviation–CAPA (corrective and preventive action) review cycles — built from existing documentation metadata without modifying validated templates.

workflow observability review interaction latency analysis streamlit python

The problem

  • Quality review cycles (deviation reports → QA review → CAPA closure) were opaque — workload imbalance, delay accumulation, and inconsistent reviewer feedback were invisible to both reviewers and submitters.
  • Teams lacked a shared visual layer for workflow coordination and bottleneck detection.

Results

N=6 (5 GMP-experienced, 1 non-GMP). All participants expressed clear intent to adopt the tool after reviewing the prototype interface.
