projects / regulatory UX / human judgment in regulated environments

How systems shape human judgment.

I study what I call Regulatory UX — the interaction experience of workers inside compliance-constrained systems, where procedural documents, audit workflows, and investigation structures organize navigation, cognition, and decision-making. These projects are the empirical ground for that argument.

The through-line across every project: In regulated environments, system design is not neutral. Document structure, workflow architecture, and AI integration all actively shape how workers search, reason, assign blame, and close investigations. Each project examines one angle of that constraint.

Research statement

I study how regulatory systems shape human judgment — through document design, workflow structure, and investigation scaffolding — in compliance-constrained GMP environments. Three core projects anchor this argument. Supporting projects extend it into adjacent failure modes.

Primary Evidence

These three projects form the empirical backbone of the Regulatory UX argument. Each addresses a distinct mechanism by which regulated systems constrain human judgment — causal reasoning, cognitive navigation, and information access.

core project 01 · lead

RCA Reasoning Scaffolder

decision support paper-track

A reasoning-support system for root cause investigations, designed to keep causal hypotheses open — countering the structural pressure toward premature blame attribution that existing documentation systems produce.

reasoning design causal mapping investigation support blame language analysis qualitative evaluation human-centered systems

The problem

  • RCA narratives converge on single causes and individual responsibility — not because evidence demands it, but because document structure does.
  • "Operator error" language appears at disproportionate rates, closing off systemic explanations before they are explored.
  • Investigators lack an external layer for maintaining and comparing alternative causal hypotheses.

Design response

A scaffold using causal diagrams, structured prompts, and hypothesis branching to support broader causal exploration. AI is positioned not to answer, but to expand the reasoning space investigators can work within — the direct design counterargument to premature accountability convergence.

AI layer · in development

Without AI: Static scaffold — causal diagram templates, structured prompts, hypothesis branching worksheets.

With AI: Dynamic reasoning partner that flags premature closure, surfaces alternative causal pathways, and detects blame-convergent language in real time.

Theoretical grounding

Directly instantiates the PAC framework (Premature Accountability Convergence) — examining what happens when AI is designed to expand interpretive space rather than stabilize responsibility. Connects to Karen Levy's research on algorithmic accountability and worker judgment.

core project 02

Cognitive Pathways in Regulated Work Environments

cognition case study

A project mapping how workers actually navigate procedural environments — revealing where formal workflow structure diverges from real cognitive pathways, and where that divergence produces friction, error, and invisible workarounds.

cognitive HCI decision traceability field observation interviews Excel Python network analysis

The problem

  • Procedural order rarely matches actual reasoning order — workers loop, revisit, and reinterpret steps under time and audit pressure.
  • Implicit decision-making is invisible to the system; only formal outputs are recorded, not how they were reached.
  • Bottlenecks cluster around cross-references, terminology ambiguity, and validation anxiety — not task complexity.

Design response

Observation, interviews, activity logging, and computational path visualization combined to surface where procedural structure conflicts with real cognitive navigation — and where targeted design interventions could reduce friction without compromising compliance integrity.

AI layer · in development

Without AI: Cognitive pathway maps showing divergence between formal and actual reasoning sequences.

With AI: Adaptive guidance that detects navigation friction in real time and surfaces relevant procedural context at decision points — reducing lookup burden without modifying validated files.

core project 03

Enhancing SOP Accessibility in GMP Environments

accessibility paper-track

A human-centered document UX project improving retrievability in scanned SOP archives — demonstrating that usability improvements can emerge through external support layers without modifying validated source files.

document UX information architecture Excel Python R usability testing · N=12

The problem

  • Scanned SOP PDFs were unsearchable and unlinked — analysts relied on manual scrolling and ambiguous file titles.
  • Retrieval delays averaged 6–7 minutes, compressing the cognitive resources available for actual compliance judgment.
  • Information access failure is a judgment failure: analysts who cannot find the right procedure cannot apply it correctly.

Outcome

Searchable metadata layer built for approximately 500 SOPs. Retrieval time reduced from ~6–7 minutes to ~1–2 minutes. N=12 usability study confirmed accessibility gains with full GMP compliance preserved. Deployed at sora-sop-demo.streamlit.app.

AI layer · in development

Without AI: Keyword-based metadata search and structured retrieval layer over validated SOP archive.

With AI: Semantic search and contextual document surfacing — retrieving not just the queried SOP, but related procedures, applicable deviations, and cross-referenced records.

Adjacent Problems

These projects extend the Regulatory UX argument into traceability, review interaction, and workflow observability — adjacent failure modes within compliance-constrained GMP environments.

supporting project 01

Human-Centered Review Interaction in Deviation & CAPA Management

interaction paper-track

A workflow interaction project focused on review clarity, feedback consistency, and turnaround time in validated deviation–CAPA documentation systems.

workflow UX review interaction Excel Python-ready survey design latency mapping

What it addressed

  • Different QA reviewers applied inconsistent feedback styles, creating unpredictable revision burdens.
  • Redundant identifier entry created friction during revision loops.
  • Small clarifications still triggered multi-day review delays.

Outcome

Review interaction framework using latency mapping, reviewer checklists, heatmaps, and usability-oriented workflow analysis to improve clarity while preserving validation integrity.

supporting project 02

Human-Centered Cross-Reference System for GMP Documentation

traceability case study

A metadata-driven cross-reference system improving document traceability and reducing cognitive load across deviation, CAPA, and related GMP records.

traceability metadata mapping Excel Python R workflow visualization

What it addressed

  • Validated files could not be automatically interlinked — users manually cross-referenced IDs, status, and closure records.
  • Version inconsistency and lookup burden weakened inspection readiness.

Outcome

External cross-reference matrix plus computational linkage analysis, enabling faster tracking and lower version-error risk without modifying validated systems.

supporting project 03

Smart Feedback & Review Dashboard

analytics prototype study

A compliance-safe workflow observability project visualizing reviewer workload, latency, and bottlenecks using existing documentation metadata.

workflow observability dashboard Excel Python R latency analysis

What it addressed

  • Review cycles were opaque and discussed only informally — workload imbalance and delay accumulation were hard to detect early.
  • Teams lacked a shared visual layer for workflow interpretation and coordination.

Outcome

Observability dashboard built from timestamps and reviewer metadata, enabling earlier bottleneck recognition and more transparent coordination without touching validated templates.
