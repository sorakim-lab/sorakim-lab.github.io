// 01

Problem

In GMP environments, every Deviation, CAPA, and Change Control record must be cross-linked to demonstrate process integrity and inspection readiness. These records are deeply interdependent, but they exist as separate validated Word files that cannot be freely restructured or automatically linked.

Analysts and QA reviewers must repeatedly copy and verify identifiers across multiple records. What looks like a traceability requirement on paper becomes, in practice, a cognitive burden placed on people — friction that accumulates invisibly across hundreds of linked records.

// observed pain points
  • Users relied on memory or keyword search to link related records
  • Inconsistent file naming increased outdated version risk
  • QA reviewers spent excessive time checking status and closure consistency
  • Manual linkage work created fatigue during audits
// HCI reframing

These were not individual user mistakes. They were consequences of system design under validation constraints, where automation was limited and humans became the connectors of information across fragmented records. Each cross-reference check that should be automatic instead becomes a cognitive task.

In validation-constrained systems, people often become the missing infrastructure between documents that cannot easily talk to each other.
// 02

Method

01
synthetic dataset construction

Generated a synthetic GMP documentation network of 150 records (60 Deviations, 40 CAPAs, 30 Change Controls, 20 SOPs) replicating structural properties of real regulated documentation — including partial linkage, version drift, and lifecycle inconsistency.

02
cross-reference metadata mapping

Extracted key identifier fields — Deviation ID, CAPA ID, batch number, product code, status, closure date, SOP version — and constructed a structured linkage matrix with status-based filters and keyword indexing.

03
computational linkage analysis + network graph

Used Python (pandas, networkx) to automate cross-link verification, detect missing connections, identify outdated SOP references, and surface lifecycle mismatches. Modeled the documentation network as a directed graph — visualizing hub concentration, isolated clusters, and cognitive load distribution.

04
interactive prototype + workplace evaluation · N=17

Built a Traceability Explorer in Streamlit and evaluated it with 17 GMP QC practitioners using real workplace data and structured traceability tasks.

The network graph reveals density asymmetry (hub records concentrating cognitive work), isolated clusters (traceability gaps invisible until audit pressure), and 3.71 average linkages per record. Each edge in the graph is a manual lookup — each disconnected node is a gap that only a person can fill.

Without computational analysis, the traceability problem can only be described anecdotally. The network graph makes cognitive burden visible — and therefore measurable.

// 05

Usability Study

N=17 GMP QC practitioners using real workplace data. Three structured traceability tasks: information retrieval, document linkage, and path tracing. Measured task completion time, document discovery, path completion rate, and navigation efficiency.

// task design
  • Task 1 — Retrieval: "Find the SOP relevant to this situation."
  • Task 2 — Linkage: "Find the documents related to this SOP."
  • Task 3 — Path tracing: "Describe how you moved from the initial document to a final judgment."
// measurement
  • Task completion time (seconds)
  • Number of documents discovered
  • Path completion rate — meaningful traceability chain established
  • Click count / navigation efficiency
  • Ease of use and real-world adoption intent
// 06

// 07

Paper in Progress

This project is being developed into a paper on traceability, metadata infrastructure, and cognitive load in compliance-constrained documentation systems. The central argument: traceability is not only a technical requirement but also an interaction and cognition problem shaped by documentation architecture — and lightweight external support layers can make that work visible without breaching system integrity.
