SOP Accessibility System

document UX python streamlit usability study

In pharmaceutical manufacturing, Standard Operating Procedures define how every critical task is performed. When those documents are hard to find, friction travels downstream into hesitation, workarounds, and judgment under constraint. This project builds a compliance-safe retrieval layer that treats document accessibility as an HCI problem.

This project does not claim to measure responsibility outcomes directly, but isolates accessibility as an upstream condition that structures interpretive pathways.

retrieval difficulty → interpretive variance → judgment under constraint → perceived responsibility

// 01

Problem

Procedural archives are often stored as scanned PDFs — blocking keyword search, burying relevant sections behind similar-sounding titles, forcing users to rely on folder memory. The problem is not missing functionality. It is a structural mismatch between how archives are organized and how people actually search under constraint.

// observed friction points
  • Scanned files blocked keyword search
  • Titles too broad, similar, or generic
  • Users opened multiple wrong files before locating target
  • Access required archive naming familiarity
  • Time pressure amplified every friction point
// structural implication

Accessibility problems can introduce hesitation and interpretive variance before any formal task error occurs. What looks like individual failure downstream often begins as structural difficulty upstream.

// 02

Method

01
metadata & keyword mapping

Reviewed approximately 500 procedural documents. Extracted keywords, document identifiers, page references, and process categories to build a searchable metadata index.

02
structure & overlap analysis

Used Python to inspect overlapping terms, recurring categories, and keyword patterns — identifying where users face ambiguity from similar titles and mismatched labels.

03
prototype development

Built an external retrieval layer allowing keyword-based search without altering validated source files. Compliance-safe by design — no modifications to any controlled document.

04
usability study · N=12 · think-aloud protocol

Used the prototype as both a design intervention and an analytic probe — observing how documentation access shapes confidence, search behavior, and navigation experience under realistic task conditions.

// 03

UI Prototype

A lightweight keyword-based search interface for procedural document retrieval. No modifications to validated source files.

Open Demo ↗
search interface · demo video

Keyword entry replaces filename memory — users enter task terms, not document titles.

Constraint
Validated SOP files cannot be modified — any change requires full revalidation under GMP regulations.
Users rely on folder memory and trial-and-error file opening, creating interpretive friction before any task begins.
Decision
External retrieval layer — search operates over metadata index, not source files.
Keyword-based access decouples retrieval from archive structure while preserving document integrity.
Trade-off
No automation of interpretation or compliance decisions — human interpretive authority is preserved.
System surfaces documents; practitioners retain full responsibility for reading, applying, and judging.
// 04

AI Support Layer

Retrieval is only one component of procedural usability. A conceptual AI-assisted extension explores how retrieval cues could be expanded — making visible why results appear, what adjacent procedures matter, and where ambiguity remains. It does not replace reading or make compliance decisions.

Conceptual extension — no separate usability study conducted on this layer. Included as a design direction, not an empirical claim.

Open AI Layer ↗
AI support layer · demo video

Three design principles made explicit: explains why a result appeared, surfaces related procedures, keeps interpretive options open.

This prototype suggests that retrieval is only one component of procedural usability, and that interpretive support is necessary to maintain reasoning openness in regulated environments.
// 05

Usability Study

This study shows: participants initially relied on folder memory and trial-and-error file opening, while the prototype enabled direct keyword-based retrieval — observed reduction in retrieval time (~6 → ~1 min, informal timing, exploratory measure). As retrieval effort decreased, participants appeared to allocate more attention to interpretation rather than navigation.

// 06

What This Revealed

This study shows that accessibility constraints shape how practitioners navigate and interpret procedural material under time pressure.

This work suggests that accessibility may function as an upstream condition — potentially influencing how responsibility is interpreted when retrieval fails or is delayed.

// 07

Paper

Claim

This paper argues that document inaccessibility in regulated environments can create conditions for recurring judgment errors — extending beyond compliance gaps into interpretive structure.

What it does
Develops a prototype-based study isolating accessibility as an upstream condition that shapes interpretive variability in SOP systems — arguing that compliance-safe usability improvements can be achieved through external support layers.
